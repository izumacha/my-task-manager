# Any Planner — タイムライン型タスクプランナー

> 1 日のタスクを「時間割」のように並べて、空き時間をひと目で把握できるデスクトップアプリです。

[Any Planner](https://apps.apple.com/jp/app/any-planner-%E3%82%BF%E3%82%B9%E3%82%AF%E7%AE%A1%E7%90%86-todo%E3%83%AA%E3%82%B9%E3%83%88/id6758033935) を参考にした、Python + tkinter 製の GUI アプリです。
**起床〜就寝の時間軸（タイムライン）にタスクを置き、空いている時間をそのまま「余白」として見える化**します。
今すぐやらないことは「あとでやる」リストに退避し、空き時間に収まるものを提案。開始時刻になると通知でお知らせします。

このアプリならではの特徴が **「完了時点からの繰り返し」**。掃除や運動のように「やったタイミングから一定間隔あけたい」タスクを、予定日ではなく*こなした瞬間*を起点に再スケジュールします（→ [詳しくはこちら](#完了時点からの繰り返しとは)）。

## クイックスタート

Python 3.10 以上が必要です。tkinter は標準ライブラリのため、追加インストールは基本的に不要です（一部の Linux では `python3-tk` パッケージが必要な場合があります）。

```bash
# 1. リポジトリを取得
git clone https://github.com/izumacha/my-task-manager.git
cd my-task-manager

# 2. 依存パッケージをインストール（ウィンドウアイコン表示用の cairosvg のみ）
pip install -r requirements.txt

# 3. 起動
python -m reminder
```

> **アイコンが無くてもOK**: `cairosvg` はウィンドウアイコン表示に使うだけのオプション依存です。入っていなくてもアプリは問題なく動きます。

### pipx でコマンドとしてインストール（任意）

仮想環境を意識せずに `reminder` コマンドで起動したい場合は [pipx](https://pipx.pypa.io/) が便利です。

```bash
pipx install git+https://github.com/izumacha/my-task-manager.git
reminder
```

## このアプリでできること

| できること | 概要 |
|---|---|
| **今日のカレンダー（デイビュー）** | 起床〜就寝を縦の時間軸に取り、タスクを*所要時間ぶんの高さ*を持つカードとして配置。空き時間は「カードの無い余白」として見えます。 |
| **あとでやるリスト** | 時間を決めていないタスクを保管。今の空き時間に収まる候補を緑色で提案します。 |
| **完了時点からの繰り返し** | なし / 日 / 週 / 月 / 年 を間隔（1〜99）付きで指定。完了した瞬間を起点に次回を自動再登録します。 |
| **通知** | 開始時刻になるとダイアログ＋通知音／デスクトップ通知でお知らせ（OS ごとに最適な方法を自動選択）。 |
| **統計** | 今日の完了件数・連続達成日数（ストリーク）・本日の空き時間をヘッダに表示。 |
| **自動保存・繰り越し** | タスクと設定は自動保存。未完了のまま日付をまたいだタスクは当日へ繰り越し、古い完了タスクは自動整理。 |

## 画面の見方

メイン画面は「今日のカレンダー（デイビュー）」です。1 日を縦の時間軸として描き、各タスクを Any Planner 風のカードで表示します。

- **タスクカード** … カテゴリ色の左ストライプ＋丸いチェックボックス＋タイトル＋時刻で構成。高さは所要時間に比例します。
- **罫線と時刻ラベル** … 1 時間ごとの罫線（30 分の薄い補助線つき）。起床が `07:30` のような半端な時刻でも、罫線は実時計の正時（08:00…）に合わせて引かれます。
- **現在時刻ライン（now ライン）** … 「今この瞬間」を横線で表示します。
- **空き時間** … カードの無い余白がそのまま空き時間です。

### 状態による色分け

タスクの状態が色で分かります。

| 状態 | 色 |
|---|---|
| 完了 | グレー（✓ 付き） |
| 進行中 | ティールグリーン |
| 過去なのに未完了 | 赤 |
| これから | カテゴリ色 |

> 起床前・就寝後に始まる/終わるタスクも見切れずに表示・操作できるよう、表示範囲は実際のタスクを内包するよう自動で広がります。

## 基本的な使い方

### タスクを追加する

1. **タスク名**を入力します。
2. **開始**の「時」「分」と**所要(分)** を選びます。
3. 繰り返したい場合は **繰り返し** で 日/週/月/年 と間隔（1〜99）を指定します。
4. **タイムラインへ**（または Enter キー）で今日の時間軸に配置されます。
   - 時間を決めずに保管したいときは **あとでへ** を押して「あとでやる」リストに追加します。

### あとでやるリストを活用する

- 「あとでやる」のタスクは、今の空き時間に収まるものが緑色で提案されます。
- **予定に追加**で開始時刻を付けてタイムラインへ移動、**あとでへ**で逆方向に戻せます。

### タスクを完了・削除する

- タイムライン上では、カードの**チェックボックスをクリックで完了**、**カード本体をクリックで選択**（再クリックで解除）。
- 選択したタスクは **完了** / **削除** ボタンで操作できます。
- **繰り返しタスクを完了すると、完了した瞬間を起点に次回タスクが自動登録**されます。

### 生活リズムに合わせる

- ヘッダの **起床／就寝** 時刻を変えると、タイムラインの表示範囲と空き時間の計算が更新されます（設定は自動保存）。

## 「完了時点からの繰り返し」とは

一般的なリマインダーは「元の予定日」を基準に次回を決めます。本アプリは **タスクを完了した日時** を基準に次回期限を算出します。

例えば「掃除（毎週）」を予定より 2 日遅れで終わらせても、次回は*完了した日*から 1 週間後に設定されます。
**こなしたタイミングから一定間隔をあけたいタスク**（掃除・運動・水やり・支払いなど）に向いています。

| 単位（間隔） | 例：6/6 22:00 に完了した場合の次回 |
|---|---|
| 日（1） | 6/7 22:00 |
| 週（2） | 6/20 22:00 |
| 月（1） | 7/6 22:00（月末日は短い月の末日にクランプ） |
| 年（1） | 翌年 6/6 22:00（2/29 は平年 2/28 にクランプ） |

## データの保存場所とバックアップ

タスク一覧と設定（起床/就寝時刻・完了履歴）は自動で保存・復元されます。

- **Linux**: `~/.config/reminder/tasks.json` / `~/.config/reminder/settings.json`
- **macOS / Windows**: 同等のユーザーディレクトリ配下に保存します（詳細は `reminder/config.py` を参照）。

### バックアップ

退避したい場合は両ファイルをコピーするだけです。

```bash
cp ~/.config/reminder/tasks.json    ~/.config/reminder/tasks.json.bak
cp ~/.config/reminder/settings.json ~/.config/reminder/settings.json.bak
```

### 壊れにくい設計

- 将来キーが増えても、未知のキーは無視されます。
- 壊れたタスクエントリが 1 件あっても、残りのタスクは読み込まれアプリの起動を妨げません。

## デスクトップアプリ化（Linux）

ターミナルを開かず、アプリ一覧（ランチャー）から起動できるようにできます。

```bash
pip install -r requirements.txt   # まだの場合
./install_reminder_app.sh
```

実行すると以下が作成され、アプリ一覧に **Any Planner** が表示されます。

- デスクトップエントリ: `~/.local/share/applications/reminder.desktop`
- 起動用ランチャー: `~/.local/bin/reminder-app`
- アイコン: `assets/reminder_icon.svg`

> アプリ一覧に表示されない場合は、一度ログアウト/ログインするか、デスクトップ環境のキャッシュを更新してください。

## 開発者向け情報

### セットアップとテスト

```bash
pip install -r requirements.txt        # 実行用
pip install -r requirements-dev.txt    # 開発・テスト用（pytest を含む）

python -m pytest tests                 # テスト実行
```

ロジック（繰り返し計算・タイムライン構築・統計など）は GUI 非依存の純粋関数として実装しているため、tkinter なしでテストできます。テストの観点は次のとおりです。

| テストファイル | 検証内容 |
|---|---|
| `test_recurrence.py` | 完了時点からの繰り返し計算（日/週/月/年・間隔）、月末/うるう年クランプ、ラベル⇔単位変換 |
| `test_timeline.py` | タイムライン構築（空き時間・状態・重なり）、起床/就寝境界、当日繰り越し、完了整理、空き時間提案、時刻整形 |
| `test_stats.py` | 今日の完了件数・連続達成日数・総完了数、不正エントリの無視 |
| `test_task.py` | Task の既定値・所要時間クランプ・あとでやる・直列化、`make_due`、`build_next_task` |
| `test_config.py` | タスク/設定の永続化・読み込み、ファイル欠損・非リスト・壊れたエントリ・不正 JSON の処理 |
| `test_notifications.py` | `play_notification_sound` の各 OS パス、`TclError` の無視、`notify-send` 本文、`_set_window_icon` |
| `test_planner.py` | 入力正規化、タイムライン/あとでへの追加・完了・削除・移動、通知スケジュール、`main` |
| `test_theme.py` | デザイントークンとカテゴリ色割り当て |

### 設計の考え方

- **ロジックと UI を分離**: ビジネスロジックは純粋関数（`timeline.py` / `stats.py` / `recurrence.py` / `task.py`）として保ち、表示層（`app.py`）を差し替えても再利用できます。
- **デザイントークンの一元化**: 配色・フォント・余白・カレンダー寸法は `reminder/theme.py` に集約。ここを編集するだけで全体の見た目を調整できます。
- **クロスプラットフォーム対応**: 通知などの OS 依存処理は `platform.system()` で分岐し、必ずフォールバックを用意（macOS=`afplay` / Windows=`winsound` / Linux=`notify-send`+`tk.bell()`）。

### ファイル構成

```
my-task-manager/
├── reminder/                   # タスクプランナーアプリ パッケージ
│   ├── __init__.py             # パッケージ公開 API
│   ├── __main__.py             # エントリーポイント (python -m reminder)
│   ├── cli.py                  # main()（コンソールスクリプト本体）
│   ├── app.py                  # PlannerApp GUI クラス（カレンダー Canvas + あとでやる）
│   ├── task.py                 # Task モデル（開始/所要/繰り返し）・次回タスク生成
│   ├── timeline.py             # 1 日のタイムライン構築・空き時間・繰り越し（純粋ロジック）
│   ├── stats.py                # 完了数・連続達成日数の集計（純粋ロジック）
│   ├── recurrence.py           # 完了時点からの繰り返し計算（日/週/月/年）
│   ├── config.py               # タスク一覧・設定の永続化 (JSON)
│   ├── notifications.py        # 通知音・デスクトップ通知・アイコン設定
│   ├── time_utils.py           # 開始までの遅延計算・定数
│   └── theme.py                # デザイントークン（配色・フォント・余白）の一元定義
├── install_reminder_app.sh     # Linux 向けデスクトップエントリ生成スクリプト
├── requirements.txt            # 実行用依存（cairosvg）
├── requirements-dev.txt        # 開発・テスト用依存（pytest）
├── pyproject.toml              # パッケージ定義・console-scripts
├── assets/
│   └── reminder_icon.svg       # アプリ用アイコン
└── tests/                      # ユニットテスト（モジュール別）
```

より詳細な設計方針やモジュールごとの責務は [`CLAUDE.md`](CLAUDE.md) にまとめています。

### 配布（参考）

Python 未導入の環境にも配布したい場合は、`pyinstaller` で単体実行ファイル化する方法があります（バイナリが大きくなる・OS ごとにビルドが必要という難点はあります）。

`reminder/__main__.py` は相対 import（`from .cli import main`）を使うため、PyInstaller の入口スクリプトに直接渡すと失敗します。代わりに、絶対 import で `main()` を呼ぶ 1 行のランチャーを用意します。

```bash
pip install . pyinstaller
echo "from reminder import main; main()" > launch_reminder.py
pyinstaller -F -w -n reminder launch_reminder.py
```
